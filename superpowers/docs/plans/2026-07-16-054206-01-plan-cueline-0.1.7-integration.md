## Recovery and stop conditions

- Before publication, rollback is the untouched `main` plus the merge commit immediately before the failing branch. Never reset, force-push, or discard an experiment branch.
- Stop before the next merge on any unresolved test failure, contract contradiction, ambiguous conflict resolution, secret exposure, or runtime mismatch.
- Stop before publication if `0.1.7` already exists on npm or GitHub, authentication is unavailable, candidate/publish preflight fails, or the frozen tarball changes.
- After npm publication, never replace `0.1.7`; release a new patch only after repairing and re-verifying.
